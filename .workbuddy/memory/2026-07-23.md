### 关键技术点（可复用）
- RimWorld 1.3 游荡精神状态行为由 think tree 的 `ThinkNode_ConditionalMentalStates` **按 defName** 门控（非类类型）；自定义 def 必须 XML patch 加入该 `<states>` 才生效。
- `JobGiver_WanderOwnRoom` 按 `pawn.MentalState is MentalState_WanderOwnRoom`（类类型）判定 → 复用 stateClass 即可。
- `TryStartMentalState(forced:true)` 绕过 `StateCanOccur`（如卧室限制）；PostStart 无卧室时 fallback 到 pawn.Position。
- 固定时长：minTicksBeforeRecovery==maxTicksBeforeRecovery 且 recoveryMtbDays=-1 → 精确按 age 计时恢复。

## 失眠发作新增保护逻辑：休息值 > 80% 不触发（22:50）
- `Need_Cortisol.TryTriggerInsomniaBreak()` 在「存在神经衰弱 Hediff」判定之后、InMentalState 判定之前，新增保护：`if (pawn.needs?.rest != null && pawn.needs.rest.CurLevelPercentage > 0.8f) return;`。
- 设计含义：已充分休息（rest>80%）时不触发失眠发作；因睡眠期间 rest 本就 >80%，故失眠发作不再打断睡眠（forceWake 实际只在清醒且 rest≤80% 时生效），更贴合“失眠=疲惫却睡不着”的语义。
- 编译：`dotnet build Assembly-CSharp.csproj -c Release` → 0 错误（仅 3 个已有的 CS0618 过时警告）。DLL 输出 `_indexProj/bin/Release/net48/RimHormones.dll`。
- 部署：XML(Defs/Patches) 已 cp 到 Steam 模组；DLL 因游戏进程(RimWorldWin64.exe)占用锁定未能覆盖，需**先关闭游戏**再 cp/跑 copyToRimWorld.bat，之后完全重启生效。
- 附：关于“总在睡醒后才发作”的排查——经查 `MentalStateHandler.TryStartMentalState`，`forceWake:true` 会 `EndCurrentJob(InterruptForced)` 强制唤醒，且 `NeedsTrackerTickInterval` 无睡眠门控，皮质醇主循环睡眠中也照常跑；故触发逻辑本不依赖睡眠，之前多为观察/旧DLL残留所致。
