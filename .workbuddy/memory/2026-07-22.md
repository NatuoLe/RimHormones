
# 2026-07-22 皮质醇 Need 数值/飘字修复

## 用户日志反馈
日志出现 `[Cortisol-飘字检查] Nickle: CurLevel=0.0 ...` => Need 已实例化并 tick（零日志问题已解决，是部署/重开问题）。
但 CurLevel 恒为 0.0、变化量 0.0、飘字阈值 2.0 永远达不到 => 暴露数值尺度 bug。

## 根因（经 rimsage 确认 NeedInterval 每 150 tick 调用一次 IsHashIntervalTick(150)）
1. GetDecayPerTick/GetGrowthPerTick 末尾 `return x / 60000f * MaxLevel`：
   - 常量(CortisolDecayNormal=13)已是 CurLevel(0-100) 刻度的占比数值，再乘 MaxLevel(100) 尺度错乱；
   - 且每个 NeedInterval 只应用了 1 tick 量，没乘 150 间隔 => 实际速率严重不对。
   改：`return x * 150f / 60000f;`（每 150 tick 区间量），并改名为 GetDecayPerInterval/GetGrowthPerInterval。
2. 飘字阈值写死 2.0（CurLevel 单位=2%严重度），而每 150 tick 真实变化仅零点零几 => 飘字永不触发。改为 1.0。
3. TryTriggerNeurastheniaCheck 每个 NeedInterval 都掷骰（约400次/日），概率被放大。加 ticksSinceLastNeuroCheck 节流，每 6000 tick 才掷一次。
