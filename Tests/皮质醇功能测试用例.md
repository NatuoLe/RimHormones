## 0. 测试环境与通用操作

### 0.1 如何在游戏内驱动测试
- **设定皮质醇浓度**：开发模式（Dev）→ 选中 pawn → `Health` 面板 → 找到 `Cortisol` hediff → 直接拖 `Severity` 条，或给 pawn 加/减 `Cortisol` hediff 后调 severity。
- **观察日志**：所有浓度变化每 60 tick 打印一条 `[皮质醇-更新] ... 触发源: ...`，可直接看哪几个应激源在生效。
- **设定其他 need**：Dev 模式给 pawn 调 `food` / `beauty` / `mood` 的 CurLevel，或加 `Pain` 类 hediff 制造疼痛。
- **快进时间**：用游戏速度 3× 或 Dev 的 `Tick once` 多次，也可直接拉 `Cortisol` 的 severity 触发档位相关逻辑。

## 8. 已知偏差 / 未完成项（测试前必读）

下列为「需求意图」与「当前代码」不一致之处，测试时请勿把旧行为当成 bug 误报，也记录为待重构项：

| 项 | 需求意图 | 当前代码实际 | 影响测试 |
|----|----------|--------------|----------|
| 档位枚举 `CortisolLevel` | 3 档：0.33 / 0.66 | 仍是 4 档：0.15 / 0.5 / 0.75（SleepDeprivation/Normal/ChronicStress/Overload）| `GetCortisolLevel` 返回的档位名与预期不符（但 `GetDecay` 已用 0.33/0.66，正确）|
| 过载耗竭机制 | 删除 | `HandleOverloadExhaustion` / `ApplyOverloadExhaustion` / `CortisolOverloadExhaustion` def 仍在 | TC-060/061 仍会通过，但属应删代码 |
| 神经衰弱概率 | 按档位 0% / 3% / 8% | 旧公式 `0.15+0.03×S×(13-P)` 封顶 0.6 | TC-031 实测为旧值 |
| 检测间隔 | 6000 tick | `CortisolDailyCheckInterval = 60000` | 神经衰弱判定周期仍是 1 天 |
| 睡眠效率 | 睡眠恢复 -33%（RestRateMultiplier 0.67）| 用的是 `RestFallRateFactor 1.33`（语义=更易疲劳）| TC-036 当前测到的是"更易疲劳"而非"睡不好" |
| ⚠️ 单位疑似错位 | 增长/衰减需同单位 | 增长项 ×60（每 60-tick 周期），衰减项是每-tick 却每周期只减一次 | 绝对数值可能过快累积，建议单独做平衡验证（见备注）|

### 备注：平衡验证建议
当前 `ProcessCortisolTick` 中，增长源返回值已乘 60（按"每秒"再 ×60tick），而 `GetDecay` 返回的是**每 tick** 值却在每 60-tick 周期里只减一次。两套单位不一致，会导致浓度上升速度远大于设计意图（可能数秒就拉满）。建议在 TC-001/TC-002 之外**单独跑一条平衡用例**：记录 60000 tick（1 游戏日）内的实际 severity 净变化，对比设计预期的"每日 13/8/3%"是否符合，必要时统一单位（增长也改为每-tick 基准，或衰减乘 60）。
